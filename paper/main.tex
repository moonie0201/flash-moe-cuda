\begin{abstract}
Mixture-of-Experts (MoE) models achieve strong performance by activating only a small subset of experts per token, but their massive weight files create severe I/O bottlenecks on consumer hardware. We describe a system for running Qwen3.5-397B-A17B (397 billion parameters, 120GB on disk at 2-bit quantization) on two consumer NVIDIA RTX 3060 GPUs (24GB total VRAM) with 96GB DDR4 RAM. Our 3-tier expert cache---GPU VRAM (57\% hit), CPU pinned RAM (43\% hit), SSD (0.1\% miss)---achieves \textbf{1.35 tokens/second} at K=10. Reducing active experts to K=4 reaches \textbf{1.80 tokens/second} at the cost of output quality. We systematically evaluate prediction-based prefetching (n-gram, MLP, FATE gate weights), and find that all approaches are bounded by the same PCIe Gen3 ceiling of 13~GB/s, explaining why caching dominates prediction for discrete GPU systems. Apple Silicon's 400~GB/s unified memory eliminates this ceiling entirely, achieving 4.4~tok/s on the same model as a reference point.
\end{abstract}

% ===================================================================
\section{Introduction}
% ===================================================================

Large Mixture-of-Experts (MoE) models~\cite{shazeer2017outrageously,fedus2022switch} achieve competitive performance while activating only a small fraction of their parameters per token. Qwen3.5-397B-A17B~\cite{qwen3.5} contains 397 billion total parameters across 512 experts per layer, with only 10 experts (K=10) activated per token. This sparsity enables on-device inference through SSD streaming, but creates a severe I/O problem: each token requires loading up to 600 expert weight tensors ($K \times 60$ layers).

The model's total expert weights are 120GB at 2-bit quantization (3.38MB per expert), far exceeding the 24GB combined VRAM of two consumer RTX 3060 GPUs. Prior work on Apple Silicon (48GB unified RAM, 17.5~GB/s SSD) streams directly from SSD. Consumer desktop hardware reverses these constraints: slower SSD ($\sim$3.5~GB/s) but more RAM (96GB), creating a different optimization landscape.

\textbf{Key question}: Can a 3-tier cache hierarchy (GPU VRAM + CPU RAM + SSD) overcome the slower SSD and PCIe bandwidth bottleneck of discrete GPU systems?

Our contributions:
\begin{enumerate}
    \item \textbf{3-tier expert cache}: GPU VRAM (6.8GB, 1980 experts, 57\% hit) + CPU pinned RAM (30GB, 9216 experts, 43\% hit) + SSD (0.1\% miss). Combined 99.9\% cache hit rate.
    \item \textbf{Float8 non-expert quantization}: Reduces non-expert weight VRAM from 19.6GB to 12.0GB (freed 7.64GB), enabling a larger GPU expert cache.
    \item \textbf{K=4 speed mode}: Reducing active experts from 10 to 4 reduces expert loads from 600 to 240 per token, achieving 1.80~tok/s with a quality trade-off.
    \item \textbf{Prediction ceiling analysis}: All prefetch approaches (n-gram, MLP, FATE gate prediction) hit the PCIe Gen3 ceiling (13~GB/s CPU$\to$GPU), confirming that caching capacity---not prediction accuracy---is the binding constraint on discrete GPU systems.
\end{enumerate}

\section{Related Work}
% ===================================================================

\subsection{Expert Offloading and Prefetching}

Expert offloading stores expert weights in CPU memory or SSD to overcome VRAM constraints~\cite{xue2024moeinfinity}. Several systems use prediction-based prefetching: FATE~\cite{fate2025} achieves 97\% prediction accuracy using cross-layer gate similarity ($W^{gate}_{l+1} \cdot h_l$). PreScope~\cite{prescope2025} combines learnable predictors with asynchronous I/O for 141\% throughput improvement. Our work shows that prediction accuracy improvements are negated by PCIe bandwidth contention on discrete GPU systems---a fundamental architectural constraint absent in unified memory systems.

\subsection{CPU/GPU Hybrid Inference}

KTransformers~\cite{ktransformers2025} uses CPU/GPU hybrid MoE inference with AMX-optimized kernels. llama.cpp supports partial GPU offloading for MoE models but does not implement multi-tier expert caching.

\subsection{SSD-Based Inference on Apple Silicon}

Apple Silicon's unified memory (400~GB/s) and fast NVMe (17.5~GB/s) enable SSD streaming at 4.4~tok/s for 397B. Discrete GPU + PCIe Gen3 ($\sim$13~GB/s H2D) creates a fundamentally different performance ceiling.

\section{System Design}
\label{sec:method}
% ===================================================================

\subsection{Hardware and Model}

\textbf{Hardware}: 2$\times$ NVIDIA RTX 3060 (12GB each, 24GB total VRAM), 96GB DDR4 RAM, consumer NVMe SSD ($\sim$3.5~GB/s), PCIe Gen3 x16 ($\sim$13~GB/s H2D).

\textbf{Model}: Qwen3.5-397B-A17B --- 60 transformer layers (45 GatedDeltaNet linear attention + 15 full attention), 512 experts per layer, $K=10$ active experts per token, hidden dimension 4096. Expert weights: 120GB total at 2-bit (3.38MB per expert, repacked from 4-bit GPTQ).

\subsection{Layer Split}

Non-expert weights (attention projections, norms, embeddings) are distributed across both GPUs:
\begin{itemize}
    \item GPU 0: layers 0--23 (24 layers)
    \item GPU 1: layers 24--59 (36 layers)
\end{itemize}

Before float8 quantization, non-expert weights require 19.6GB VRAM across both GPUs, leaving only 4.4GB for expert cache. Float8 quantization of attention/projection weights reduces this to 12.0GB, freeing 7.64GB for expert cache.

\subsection{3-Tier Expert Cache}

\begin{figure}[t]
\centering
\begin{verbatim}
Token → routing → K experts needed
         │
         ├─ GPU VRAM  (6.8GB,  ~1980 experts) 57% hit
         ├─ CPU RAM   (30GB,   ~9216 experts) 43% hit
         └─ SSD       (120GB,  rest)          0.1% miss
\end{verbatim}
\caption{3-tier expert cache hierarchy. Total hit rate: 99.9\%.}
\label{fig:cache}
\end{figure}

\textbf{Cache population}: An initial COLD pass through the prompt builds an expert frequency map. Hot experts (most frequently activated) fill GPU VRAM first, then CPU pinned RAM. Subsequent inference passes serve expert weights from VRAM (instant) or CPU RAM (H2D transfer, $\sim$0.26ms) with only true misses touching SSD ($\sim$1.3ms).

\textbf{CPU pinned memory}: Expert tensors are allocated with \texttt{pin\_memory=True}, enabling asynchronous H2D transfers via DMA without CPU involvement.

\subsection{Fused CUDA Kernel}

Expert forward passes use a custom CUDA kernel (\texttt{expert\_ops.cu}) that fuses 2-bit dequantization + batched GEMV + SiLU activation in a single launch. This replaces 10 serial expert calls per layer with one batched launch, reducing kernel launch overhead.

\subsection{Thread Pool I/O}

SSD expert reads use a 16-worker thread pool, issuing all expert reads for a layer in parallel. This overlaps per-expert I/O latency across the $K$ experts required per layer.

\subsection{Prediction Experiments: The PCIe Ceiling}
\label{sec:prediction}

We systematically evaluated prediction-based prefetching to reduce SSD misses and CPU$\to$GPU H2D stalls:

\begin{table}[t]
\centering
\caption{Prediction-based prefetch experiments (all K=4).}
\label{tab:prediction}
\begin{tabular}{lccc}
\toprule
Method & GPU Hit Rate & FULL+GPU & vs baseline \\
\midrule
K=4 baseline & 59.9\% & 1.800 & --- \\
N-gram (per-expert top-4) & --- & 1.062 & -41\% \\
N-gram (global top-10) & --- & 1.323 & -26\% \\
MLP (1,740 samples) & 29.8\%\textsuperscript{*} & --- & n/a \\
MLP (34,800 samples) & 25.7\%\textsuperscript{*} & --- & n/a \\
FATE gate predict & 76.6\% & 1.712 & -5\% \\
FATE + shallow cache & 43.7\% & 1.497 & -17\% \\
\bottomrule
\end{tabular}
\vspace{1mm}

\textsuperscript{*}hit@10 rate on validation set; not integrated into inference loop due to poor accuracy.
\end{table}

\textbf{N-gram prefetch}: A per-expert co-activation table (82.8\% in-distribution accuracy) issued $\sim$700 H2D transfers per token vs 310 actually needed. PCIe congestion from over-prediction caused a 41\% slowdown.

\textbf{MLP predictor}: A 3-layer MLP trained on $h_{\text{normed}}[L]$ to predict expert activations at layer $L+1$ achieved 29.8\% hit@10 on 1,740 training samples, degrading to 25.7\% with 20$\times$ more data. Cross-layer prediction is fundamentally difficult: the attention + expert compute + residual + normalization between layers makes $h[L]$ a weak predictor of $L+1$ routing.

\textbf{FATE gate prediction}: Implementing the gate-weight prediction from FATE~\cite{fate2025} ($\hat{e}_{l+1} = \text{TopK}(W^{gate}_{l+1} \cdot h_l)$) improved GPU hit rate from 59.9\% to 76.6\% (prediction is working), but throughput dropped from 1.800 to 1.712~tok/s. Accurate prefetch H2D transfers compete with cpu\_hit$\to$GPU transfers over the same PCIe bus.

\textbf{Root cause}: PCIe Gen3 H2D bandwidth ($\sim$13~GB/s) is the binding constraint. Adding prefetch traffic to the bus---even perfectly accurate prefetch---adds H2D load that cancels the gain from higher GPU hit rates. All prediction approaches, regardless of accuracy, hit this ceiling.

\textbf{Trade-off}: Skipping 6 of 10 experts per token reduces output quality. Generation remains coherent for general use but is not recommended for tasks requiring model accuracy (long-form reasoning, code generation).

\section{Discussion}
% ===================================================================

\textbf{PCIe as destiny}: For discrete GPU + CPU systems, PCIe Gen3 H2D bandwidth is the fundamental bottleneck. Expert weight data ($K$ experts $\times$ 60 layers $\times$ 3.38MB = up to 2.028GB per token at cold) must traverse PCIe for every token. No software optimization---prediction, prefetch, compression, compilation---can overcome this for large-K inference without increasing RAM-resident cache size.

\textbf{Caching $\gg$ Prediction}: Our experiments consistently show that increasing cache capacity (more GPU VRAM, more pinned RAM) outperforms improving prediction accuracy. The fundamental reason: cache hits are free; prediction hits still require PCIe transfers. Only VRAM-resident data is truly zero-cost.

\textbf{Unified memory advantage}: Apple Silicon's 400~GB/s unified memory and 17.5~GB/s NVMe bypass the PCIe ceiling entirely. Expert weights move between storage and GPU at memory bandwidth, not PCIe bandwidth. This architectural difference explains why the Mac reference (4.4~tok/s) is 3.3$\times$ faster despite similar GPU compute: the bottleneck is different in kind, not just degree.

\textbf{Practical recommendation}: For 397B MoE inference on consumer hardware, the optimal configuration is 3-tier caching with K=4 for throughput (1.80~tok/s) or K=10 for quality (1.35~tok/s). Prediction-based approaches should be avoided---they add complexity without improving throughput due to PCIe contention.

% ===================================================================
\section{Conclusion}
% ===================================================================

We presented a 3-tier expert caching system for running Qwen3.5-397B-A17B on dual consumer RTX 3060 GPUs. The system achieves 1.35~tok/s at K=10 and 1.80~tok/s at K=4 (with quality trade-off) through frequency-aware caching across GPU VRAM, CPU pinned RAM, and SSD. We find that prediction-based prefetching---despite achieving 76.6\% GPU hit rates via FATE gate prediction---provides no throughput benefit due to PCIe bandwidth contention, establishing PCIe Gen3 as the fundamental ceiling for discrete GPU MoE inference. Apple Silicon's unified memory architecture eliminates this ceiling, representing an architectural advantage that software optimizations cannot fully overcome.
